# RaC Profile — GDPR v1 (reference profile)

This file is the **GDPR reference profile** of Regulation-as-Code (RaC): a worked
`SourceManifest` instance that encodes a wedge of the General Data Protection
Regulation (Regulation (EU) 2016/679) in the authored grammar. It is the
canonical example proving the grammar — a real regulation expressed as
machine-evaluable obligations.

The shapes used here (`SourceManifest`, `FactDef`, `Condition`, `Requirement`,
`Obligation`) are defined normatively in [`../../spec/grammar.md`](../../spec/grammar.md);
their evaluation semantics (applicability, satisfaction, the at-risk window,
`fresh`/`exists`/`all` quantifiers) are defined in
[`../../spec/evaluation.md`](../../spec/evaluation.md). The evidence families this
profile references are documented in [`../../spec/evidence.md`](../../spec/evidence.md) §3.

The key words **MUST**, **MUST NOT**, **SHOULD**, **SHOULD NOT**, and **MAY**
are to be interpreted as described in RFC 2119 / RFC 8174.

> **Disclaimer.** This profile is a machine-readable reference mapping of selected
> GDPR articles to evidence requirements. It is **not legal advice** and is **not
> individualised legal advice**. Applicability and sufficiency of any obligation
> to a specific organisation must be confirmed with qualified counsel.

## 1. Header

| Field | Value |
|---|---|
| `id` | `"gdpr"` |
| `version` | `"1.2.0"` |
| `atRiskWindowDays` | `30` |

`atRiskWindowDays: 30` is the warning window before a `fresh` requirement lapses:
an obligation that is currently satisfied but whose freshest receipt is within 30
days of its `maxAgeDays` boundary evaluates **at-risk** rather than satisfied (see
[`../../spec/evaluation.md`](../../spec/evaluation.md)).

### Fact schema

The applicability facts a workspace supplies. Facts are validated against this
schema before evaluation.

| Fact | Type | Flags |
|---|---|---|
| `processesPersonalData` | `boolean` | — |
| `usesAiFeatures` | `boolean` | — |
| `dpiaResidualRiskHigh` | `boolean` | `optional` |

`dpiaResidualRiskHigh` is the Art. 35(3) → Art. 36 trigger: it is `true` only when
a completed DPIA records **HIGH residual risk after mitigation**. It is `optional`
(default-absent ⇒ `false`) so existing consumers need not supply it; when absent,
`gdpr.prior-consultation` is **not-applicable**, so a low/medium-risk processor is
never false-red.

```ts
facts: [
  { name: "processesPersonalData", type: "boolean" },
  { name: "usesAiFeatures", type: "boolean" },
  { name: "dpiaResidualRiskHigh", type: "boolean", optional: true },
]
```

## 2. Obligations

### 2.1 Readable table

| id | GDPR article | Applies when | Requires | Evidence family |
|---|---|---|---|---|
| `gdpr.lawful-basis` | Art. 6 — lawfulness of processing | `processesPersonalData` | `exists` | **ACR** (Anchored Consent Receipt) |
| `gdpr.purpose-limitation` | Art. 5(1)(b) — purpose limitation | `processesPersonalData` | `exists` | **APuR** (Anchored Purpose Receipt) |
| `gdpr.retention` | Art. 5(1)(e) — storage limitation | `processesPersonalData` | `fresh ≤ 365d` | **ARR** (Anchored Retention Receipt) |
| `gdpr.privacy-notice` | Art. 13 — information to be provided | `processesPersonalData` | `fresh ≤ 365d` | **ANR** (Anchored Notice Receipt) |
| `gdpr.subject-rights` | Arts. 15–22 — data subject rights | `processesPersonalData` | `exists` | **ASR** (Anchored Subject-Rights Receipt) |
| `gdpr.erasure` | Art. 17 — right to erasure | `processesPersonalData` | `exists` | **FORGET** (proof-of-forgetting erasure) |
| `gdpr.dpia` | Art. 35 — data protection impact assessment | `usesAiFeatures` | `exists` | **AIR** (Anchored Impact Receipt) |
| `gdpr.prior-consultation` | Art. 36 — prior consultation | `usesAiFeatures` **and** `dpiaResidualRiskHigh` | `exists` (`eventType: dpia.prior_consultation_resolved`) | **AIR** (Anchored Impact Receipt) |

**Design notes (carried from the source):**

- `gdpr.subject-rights` is deliberately **not** split per-right: an ASR evidences a
  *handled request*, so requiring one per right would false-red a controller for
  rights nobody exercised. "Informed of rights" is covered separately by
  `gdpr.privacy-notice` (Art. 13 ANR).
- `gdpr.erasure` is fed by proof-of-forgetting destruction receipts (**FORGET**),
  **not** ASR. A destruction receipt is produced by retention enforcement (and on
  request), so it evidences an erasure *capability* without the false-red of an
  exists-ASR-per-right model.
- `gdpr.prior-consultation` (Art. 36) is gated on `dpiaResidualRiskHigh` so a
  low/medium-risk processor is not-applicable (never false-red), distinct from
  `gdpr.dpia` (Art. 35). It is satisfied **only** by an AIR carrying the
  `dpia.prior_consultation_resolved` event — a bare `dpia.completed` (which
  satisfies `gdpr.dpia`) must not green Art. 36; the `eventType` sub-family filter
  enforces the distinction.

### 2.2 Literal `SourceManifest`

The normative content of this profile. Transcribed from the shipped manifest.

```ts
const GDPR_MANIFEST: SourceManifest = {
  id: "gdpr",
  version: "1.2.0",
  atRiskWindowDays: 30,
  facts: [
    { name: "processesPersonalData", type: "boolean" },
    { name: "usesAiFeatures", type: "boolean" },
    // True when a completed DPIA records HIGH residual risk after mitigation —
    // the Art. 35(3)→36 trigger. Optional (default-absent ⇒ false) so existing
    // consumers need not supply it; absent ⇒ gdpr.prior-consultation is
    // not-applicable, so a low/medium-risk processor is never false-red.
    { name: "dpiaResidualRiskHigh", type: "boolean", optional: true },
  ],
  obligations: [
    {
      id: "gdpr.lawful-basis",
      regulation: "GDPR Art. 6 — lawfulness of processing",
      citationUrl: "https://eur-lex.europa.eu/eli/reg/2016/679/oj#d1e1888-1-1",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      requires: { kind: "exists", family: "ACR" },
      remediation: "Record a consent / lawful-basis receipt (ACR) for personal-data processing.",
    },
    {
      id: "gdpr.purpose-limitation",
      regulation: "GDPR Art. 5(1)(b) — purpose limitation",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      requires: { kind: "exists", family: "APuR" },
      remediation: "Declare the processing purpose with a purpose receipt (APuR).",
    },
    {
      id: "gdpr.retention",
      regulation: "GDPR Art. 5(1)(e) — storage limitation",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      requires: { kind: "fresh", family: "ARR", maxAgeDays: 365 },
      remediation: "Publish or refresh your retention schedule (ARR) at least annually.",
    },
    {
      id: "gdpr.privacy-notice",
      regulation: "GDPR Art. 13 — information to be provided",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      requires: { kind: "fresh", family: "ANR", maxAgeDays: 365 },
      remediation: "Publish or refresh your privacy notice (ANR) at least annually.",
    },
    {
      id: "gdpr.subject-rights",
      regulation: "GDPR Arts. 15–22 — data subject rights",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      // exists-ASR = "a subject-rights handling process is in place, demonstrated
      // by ≥1 handled request." Deliberately NOT split per-right: an ASR evidences
      // a *handled request*, so requiring one per right would false-red a
      // controller for rights nobody exercised. "Informed of rights" is covered
      // separately by gdpr.privacy-notice (Art. 13 ANR rights_surface).
      requires: { kind: "exists", family: "ASR" },
      remediation: "Stand up a subject-rights handling process and record it (ASR).",
    },
    {
      id: "gdpr.erasure",
      regulation: "GDPR Art. 17 — right to erasure",
      citationUrl: "https://eur-lex.europa.eu/eli/reg/2016/679/oj#d1e2606-1-1",
      appliesWhen: { kind: "fact", fact: "processesPersonalData" },
      // Fed by proof-of-forgetting destruction receipts (forget.destruction.v1),
      // NOT ASR. A destruction receipt is produced by retention enforcement (and
      // on request), so it evidences an erasure *capability* without the false-red
      // of an exists-ASR-per-right model. Distinct obligation per the corpus
      // adapter NOTE.
      requires: { kind: "exists", family: "FORGET" },
      remediation: "Perform verifiable erasure (crypto-shred) and record a proof-of-forgetting receipt.",
    },
    {
      id: "gdpr.dpia",
      regulation: "GDPR Art. 35 — data protection impact assessment",
      appliesWhen: { kind: "fact", fact: "usesAiFeatures" },
      requires: { kind: "exists", family: "AIR" },
      remediation: "Complete a DPIA for high-risk / AI processing and record it (AIR).",
    },
    {
      id: "gdpr.prior-consultation",
      regulation: "GDPR Art. 36 — prior consultation",
      citationUrl: "https://eur-lex.europa.eu/eli/reg/2016/679/oj#d1e3138-1-1",
      // Art. 36 is triggered ONLY when a DPIA shows high residual risk after
      // mitigation. Gated on dpiaResidualRiskHigh so a low/medium-risk processor
      // is not-applicable (never false-red), distinct from gdpr.dpia (Art. 35).
      appliesWhen: {
        kind: "all",
        of: [
          { kind: "fact", fact: "usesAiFeatures" },
          { kind: "fact", fact: "dpiaResidualRiskHigh" },
        ],
      },
      // Satisfied only by an AIR carrying the prior-consultation-resolved event —
      // NOT any AIR. A bare dpia.completed (which satisfies gdpr.dpia) must not
      // green Art. 36; the eventType sub-family filter enforces the distinction.
      requires: {
        kind: "exists",
        family: "AIR",
        where: { eventType: ["dpia.prior_consultation_resolved"] },
      },
      remediation:
        "Your DPIA shows high residual risk — consult your supervisory authority before processing and record the outcome (AIR dpia.prior_consultation_resolved).",
    },
  ],
};
```

## 3. Evidence binding

This profile binds the **anchors reference evidence families** —
ACR, APuR, ARR, ANR, ASR, AIR — plus the non-anchors **FORGET**
(proof-of-forgetting erasure) source. Each family's canonical name, role, and the
receipts that satisfy a `Requirement` against it are specified in the evidence
family registry: [`../../spec/evidence.md`](../../spec/evidence.md) §3.

The authored grammar these obligations are written in is defined in
[`../../spec/grammar.md`](../../spec/grammar.md); how a workspace's facts and
evidence corpus are evaluated against this manifest (applicability, satisfaction,
the at-risk window) is defined in [`../../spec/evaluation.md`](../../spec/evaluation.md).
